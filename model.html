crawler-if-db — Developer API Reference

Introduction

CrawlerIf is an embedded HSQLDB persistence library for a web-crawler image-gallery application. The single entry point is com.gerken.db.persist.PersistCrawlerIf; all database access goes through this class. JSON serialisation uses the Jettison library (org.codehaus.jettison.json.JSONObject).

To use this library as a Maven dependency, add the following stanza to your pom.xml:

<dependency>
    <groupId>org.gerken</groupId>
    <artifactId>crawler-if-db</artifactId>
    <version>1.0-SNAPSHOT</version>
</dependency>

Lifecycle

PersistCrawlerIf.open() is ref-counted: multiple callers that open the same database path share one underlying connection. Each caller must call close() to decrement the ref-count; the physical connection is released only when the count reaches zero. 

PersistCrawlerIf db = PersistCrawlerIf.open("/path/to/db");
try {
    // ... CRUD calls ...
} finally {
    db.close();
}

All tables and indexes are created automatically on the first open() call if they do not already exist.

Cursor API

Cursors are forward-only and read-only. next() returns the next bean, or null when the result set is exhausted — and automatically closes the cursor. Call close() explicitly (e.g. in a finally block) if you stop iterating early, to avoid resource leaks.

next() loop

GalleryCursor c = db.selectGallery("WHERE state = 'CA' ORDER BY added");
try {
    Gallery g;
    while ((g = c.next()) != null) {
        // process g
    }
} finally {
    c.close();
}

getAll()

Gallery[] all = db.selectGallery(null).getAll();

count()

int n = GalleryCursor.count("WHERE poster = 'alice'");

JSON / Bean Interoperability

Every bean has two constructors and an asJson() method: 

// PK constructor — non-PK fields start null
Gallery g = new Gallery("gal-001");
g.setPoster("alice");
g.setState("CA");

// JSONObject constructor — only the PK field is required in the JSON
Gallery g2 = new Gallery(someJsonObject);

// Serialise to JSON — all fields included; Java null becomes JSONObject.NULL
JSONObject json = g.asJson();

On input, non-PK fields absent from the JSON remain null. A field present but set to JSONObject.NULL is also treated as null.

Ad-hoc Queries

// prepare / execute / recycle cycle
PreparedStatement ps = db.prepare("SELECT COUNT(*) FROM Gallery WHERE poster = ?");
ps.setString(1, "alice");
db.execute(ps);
db.recycle(ps);   // returns statement to cache for future reuse

// scalar convenience
long total = db.getValue("SELECT COUNT(*) FROM Gallery");

Backup and Restore

backup(path) writes one JSON-lines file per type (e.g. Gallery.json) to the given directory. Each line is the JSON serialisation of one row. restore(path) drops and recreates all tables, then reloads from those files. Both operations run one thread per type in parallel. 

db.backup("/var/backups/crawlerif");
// later ...
db.restore("/var/backups/crawlerif");
